# 875 爱吃香蕉的珂珂 · 知识点笔记（二分答案 入门）

> 按知识点梳理，配合同文件夹 `solution.py` / `problem.md`，以及分类通用模板 `../binary_search.py` 看。
> 这题是**二分答案**套路的入门题——二分专题里最重要的升级，务必吃透。

## 1. 读题
- 题意大白话：`piles` 堆香蕉，`h` 小时吃完。珂珂选速度 `k`（根/小时），**每小时挑一堆吃，最多吃 k 根**；这堆不够 k 根就吃光、本小时剩下时间不再吃别堆（吃撑休息）。求 `h` 小时内吃完的**最小速度 k**。
- 例子 `piles=[3,6,7,11], h=8` → `k=4`：耗时 `1+2+2+3=8 ≤ 8` ✓；`k=3` 要 `1+2+3+4=10 > 8` ✗。所以 4 是能成功的最小速度。

## 2. 讲思路（面试就这么讲）
**三句话**：
1. 本质是"求满足条件的最小值"——速度越快越容易吃完，存在临界点，找第一个能成功的速度。
2. **二分答案**：在速度范围 `[1, max(piles)]` 上二分，配判定函数 `check(k)`（用速度 k 算总小时数 ≤ h 吗），找第一个让 check 成立的 k。
3. 时间 `O(n log(max))`（每次 check 扫一遍 n 堆，二分 log(max) 次），空间 O(1)。

**一题三讲**：
- 暴力：从 k=1 开始一个个试，第一个能吃完的就是答案，O(n·max)，max 可达 10^9 会超时。
- 为什么慢：k 一个个试没利用"单调性"。
- 怎么优化：k 有单调性（✗✗✗✓✓✓），二分这个范围，O(n·log max)。

## 3. ⭐ 二分答案"心法"（最重要的收获）
前几题都是"**在数组里二分找一个元素**"。这题是"**在答案的取值范围上二分**"——数组本身根本没排序。

为什么答案能二分？因为它有**单调性**：
```
速度 k:   1   2   3   4   5  ...  max
能吃完?:  ✗   ✗   ✗   ✓   ✓  ...  ✓
                    ↑ 找这个【第一个 ✓】= 最小可行速度
```
吃得越快用时越少 → 比临界点慢的全失败、≥它的全成功 → 这就是一条假想的有序 `✗✗✗✓✓✓`，**找第一个 ✓ = 找左边界 = 153 的模板**。

### 二分答案三件套（识别+套用）
1. **答案范围**：想清楚答案最小、最大能是多少。本题 `left=1`（再慢没意义）、`right=max(piles)`（速度比最大堆还快没用，一堆顶多 1 小时）。
2. **判定函数 `check(x)`**：给定一个候选答案 x，O(n) 判断"行不行"，返回 True/False。本题 `check(k)` = 用速度 k 算总小时数 ≤ h。
3. **在范围上二分**：找第一个/最后一个满足 check 的值。本题求最小可行 → 找第一个 True。

> 见到"**求最小的 X 使得……成立**"或"**最大化最小值 / 最小化最大值**"这类，基本就是二分答案。

## 4. check(k) 怎么算时间
每堆 `pile` 根、速度 k → 要 `ceil(pile / k)` 小时（**向上取整**，因为吃不满也占整一小时）。总时间 = 各堆相加，`<= h` 即成功。

**向上取整两种写法**：
- 直观：`import math` 后 `math.ceil(pile / k)`（本题用这个）。
- 整数技巧（面试爱考、无浮点误差）：`(pile + k - 1) // k`。
  - 原理：想"有余数就进 1"，先给分子加 `k-1` 再整除。`6/4`→`(6+3)//4=2`；整除 `8/4`→`(8+3)//4=2`（刚好整除不会多进）。

## 5. 主二分（和 153 同款"缩到单个幸存者"）
```python
left, right = 1, max(piles)
while left < right:          # < 不是 <=
    mid = (left + right) // 2
    if check(mid):
        right = mid          # mid 可行 -> 答案是 mid 或更小 -> 保留 mid
    else:
        left = mid + 1       # mid 不行 -> 答案更大
return left                  # 第一个可行值
```
**"求满足条件的最小值" = "✗✗✗✓✓✓ 上找第一个 ✓" = `while left<right` + `right=mid`**。和 153 找最小值是同一个骨架。

## 6. 复杂度
- 时间 `O(n log(max(piles)))`：二分 `log(max)` 次，每次 `check` 扫 n 堆。
- 空间 O(1)。

## 7. 易错点
- `right = mid` 配 `while left < right`（同 153），别写 `mid - 1`（会丢掉可能可行的 mid）。
- `right` 初值是 `max(piles)`，不是 `len(piles)` 或 `sum(piles)`（速度上限 = 最大堆，再大无意义）。
- `ceil` 别写成普通整除 `pile // k`（会把吃不满的零头漏掉，少算时间）。
- `left` 初值是 1，不是 0（速度 0 吃不动，没意义；且 `pile/0` 会除零）。

## 8. 面试官追问 Q&A（每题必有）
- **Q：为什么 `right` 取 `max(piles)` 就够，不用更大？**
  A：速度 ≥ 最大堆时，每堆都能在 1 小时内吃完，总耗时 = 堆数 n，已是最快。再大速度不会更快，所以答案不会超过 `max(piles)`，范围右端取它即可。
- **Q：判定函数为什么能保证二分正确？单调性从哪来？**
  A：`check(k)` 关于 k **单调**：k 越大，每堆 `ceil(pile/k)` 越小，总时间不增，越容易 ≤ h。所以 check 的结果是 `False...False True...True`，存在唯一临界点，二分才成立。二分答案的前提永远是"判定函数对答案单调"。
- **Q：时间复杂度里的 log 是对谁取的？**
  A：对**答案范围**取，即 `log(max(piles))`，不是 `log(n)`。这是二分答案和普通数组二分的一个区别——二分的是值域，不是下标范围。
- **Q：如果改成"求最大速度使得耗时 ≥ 某值"这类，模板要怎么改？**
  A：那是"找最后一个满足条件的值"（✓✓✓✗✗✗ 找最后一个 ✓），改成命中时 `left = mid + 1` 往右探、配 `mid = (left+right+1)//2` 防死循环，返回 `left`。即二分答案的"找右边界"版本。

## 9. 顺带建立的概念
- **二分答案 vs 普通二分**：普通二分在"已排序的数组下标"上找元素；二分答案在"答案的取值范围(值域)"上找临界值，靠一个 `check(x)` 把问题转成"✗✗✓✓"。数组可以完全无序，只要**答案对判定单调**就能用。
- **识别信号**："求最小/最大的 X 使得某条件成立"、"最小化最大值 / 最大化最小值"、答案有明确上下界且能 O(n) 验证。
- **整数向上取整**：`(a + b - 1) // b` 等价 `ceil(a/b)`，整数运算无浮点误差，工程/面试常用。
- 这题把二分专题从"在数组里查"升级到"在答案空间里搜"，是后续很多中高频难题（如 1011 送包裹、410 分割数组、LCP 12 等）的同一招。
