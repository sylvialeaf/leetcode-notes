# 35 搜索插入位置 · 知识点笔记

> 按知识点梳理，配合同文件夹 `solution.py` / `problem.md`，以及分类通用模板 `../binary_search.py`（模板 B 找左边界）看。

## 1. 读题
- 题意大白话：升序数组里找 `target`。找到了返回它的下标；**找不到返回"它该插进去的位置"**。
- 例子 `nums=[1,3,5,6]`：
  - `target=5` → 在下标 2 → `2`
  - `target=2` → 没有，插在 1 和 3 之间 → `1`
  - `target=7` → 比谁都大，插末尾 → `4`
  - `target=0` → 比谁都小，插最前 → `0`

## 2. 讲思路（面试就这么讲）
**三句话**：
1. 本质是"在有序数组里找一个数 / 找它该在的位置"。
2. 二分查找，主体和 704 完全一样；区别只在**找不到时返回插入位置**。
3. 时间 O(log n)，空间 O(1)。

**一题三讲**：
- 暴力解：从头扫，遇到第一个 `>= target` 的位置就是答案，O(n)。
- 为什么慢：没利用有序。
- 怎么优化：二分，O(log n)。

## 3. ⭐ 核心洞见（这题的钥匙）
> **插入位置 = "比 target 小的数有几个"。**

- 纯常识：比 target 小的数有 k 个 → target 就排在它们后面，即下标 k。
  - 比 2 小的有 `1`（1 个）→ 插在下标 1；比 7 小的有 4 个 → 下标 4；比 0 小的有 0 个 → 下标 0。
- 而二分里的 `left` **正好在数这个数**：`left` 从 0 开始，只有 `nums[mid] < target` 时执行 `left = mid + 1`，每次都意味着"又确认一批比 target 小的数，跳到它们后面"。
- 所以**循环结束时 `left` = 比 target 小的数的个数 = 插入位置**。

## 4. 和 704 的唯一区别（一行）
主体 `while left <= right` + 三种情况，和 704 **一字不差**。只改结尾：
```
704:  循环结束 → return -1      （报告不存在）
35:   循环结束 → return left    （返回插入位置）
```

## 5. 解法步骤
1. `left = 0`，`right = len(nums) - 1`。
2. `while left <= right`：算 `mid`，比较：
   - `nums[mid] == target` → `return mid`
   - `nums[mid] < target` → `left = mid + 1`
   - `nums[mid] > target` → `right = mid - 1`
3. 循环结束（`left > right`，两指针交叉）→ `return left`。

## 6. 复杂度
- 时间 O(log n)：每次范围减半。空间 O(1)。

## 7. 易错点
- 结尾返回 `left` 不是 `right`（`right = left - 1`，返回 right 会少 1）。
- `while` 带等号、指针 ±1，和 704 一样。
- 循环靠 `left > right`（两指针交叉错开 1）结束，这是二分结束的固定姿势，正常现象。

## 8. 面试官追问 Q&A（每题必有）
- **Q：为什么找不到时返回 `left` 而不是 `right`？**
  A：循环结束时 `left` 等于"比 target 小的数的个数"，正好是插入位置；而 `right = left - 1`，返回它会少 1。
- **Q：能不能干脆不写 `== target` 那个分支？**
  A：能。把"命中"也并进 `>=` 分支（`nums[mid] >= target` 时 `right = mid - 1`），最后统一 `return left` 一样正确——这就是通用模板里的"模板 B 找左边界 / lower_bound"。35 其实是 lower_bound 的直接应用。
- **Q：如果数组里有重复的 target，这个写法返回第几个？**
  A：本题保证无重复。若有重复，当前"命中即 return mid"会返回任意一个；要稳定返回**第一个** target 的位置，就用模板 B（命中也继续往左缩）。

## 9. 顺带建立的概念
- `pass`：占位符，"什么都不做"。函数体不能空，没写逻辑时用它占位。**若函数体只有 `pass`、没有 `return`，调用它会返回 `None`**（本题最初没填代码时输出 4 个 `None` 就是这个原因）。
- 二分的"找值"和"找插入位置/找边界"是同一套骨架，差别只在结束时返回什么——这是二分能举一反三的关键。
