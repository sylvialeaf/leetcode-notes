# 33 搜索旋转排序数组 · 知识点笔记

> 按知识点梳理，配合同文件夹 `solution.py` / `problem.md`，以及分类通用模板 `../binary_search.py` 看。

## 1. 读题
- 题意大白话：一个**原本升序**的数组，被从某点"掰断、前后两半对调"（叫**旋转**）。在旋转后的数组里找 `target` 的下标，找不到返回 `-1`。
- **硬约束**：时间必须 **O(log n)** —— 不准从头扫一遍（那是 O(n)），逼你用二分。
- 例子 `nums=[4,5,6,7,0,1,2]`（原 `[0,1,2,4,5,6,7]` 在 4/5 之间掰断）：
  - `target=0` → 下标 4
  - `target=3` → 不存在 → `-1`
- 画面感：旋转后是**两段各自升序**，中间一个"断崖"（从 7 直接掉到 0）。

## 2. 讲思路（面试就这么讲）
**三句话**：
1. 本质是"在被旋转过、不整体有序的数组里找一个数"，且要求 O(log n)。
2. 关键洞见：**砍一刀取 mid 后，mid 左右两半里至少有一半是完整升序的**。先判断哪半边升序，在那半边用普通二分判断 target 在不在，决定往哪缩。
3. 时间 O(log n)，空间 O(1)，一次二分。

**一题三讲**：
- 暴力解：`for` 扫一遍找 target，O(n)。
- 为什么慢 / 不行：题目死规定 O(log n)，扫一遍直接判负。
- 怎么优化：利用"两段各自升序"的结构，二分。

## 3. ⭐ 核心洞见（这题的钥匙）
> **无论砍哪一刀，`mid` 左右两半中总有一半是"没跨过断崖"的完整升序段。**

怎么认出来：比 `nums[left]` 和 `nums[mid]`——
- `nums[left] <= nums[mid]` → 左半段 `[left..mid]` **没跨断崖**，是升序的；
- `nums[left] >  nums[mid]` → 左半段跨了断崖（前面的数反而更大）→ **右半段** `[mid..right]` 才是升序的。

认出升序的那半边后，它范围清清楚楚（最小=段首、最大=段尾），就能用普通二分的"target 是否夹在 `[段首, 段尾]` 里"来判断 target 在不在那半边。

## 4. 决策树（每一刀按顺序问 3 个问题）
```
取 mid 后：

问题1：nums[mid] == target ？
   ├─ 是 → return mid  ✅ 结束
   └─ 否 → 继续

问题2：nums[left] <= nums[mid] ？  （左半段是不是升序）
   │
   ├─ 是（左半段 [left..mid] 升序）──→ 问题3A
   │      nums[left] <= target < nums[mid] ？  （target 在左半段）
   │         ├─ 是 → right = mid - 1（往左缩）
   │         └─ 否 → left  = mid + 1（往右缩）
   │
   └─ 否（右半段 [mid..right] 升序）──→ 问题3B
          nums[mid] < target <= nums[right] ？  （target 在右半段）
            ├─ 是 → left  = mid + 1（往右缩）
            └─ 否 → right = mid - 1（往左缩）

循环靠 left > right 结束 → return -1
```
**一句话记忆**：先认出哪半边升序 → 在升序那半边判断 target 在不在 → 在就缩到那半边，不在就去另一半边。

## 5. 走一遍例子（建立画面感）
`nums = [4,5,6,7,0,1,2]`，下标 `0..6`

**target=0**（答案下标 4）：
- 轮1 `l=0,r=6,mid=3,nums[3]=7`：左半段升序(`4<=7`)；`4<=0<7`否 → `l=4`
- 轮2 `l=4,r=6,mid=5,nums[5]=1`：左半段升序(`0<=1`)；`0<=0<1`是 → `r=4`
- 轮3 `l=4,r=4,mid=4,nums[4]=0`：命中 → return 4 ✅

**target=3**（不存在 → -1）：
- 一路 `left` 往右推，最终 `left=7 > right=6`，`while` 不成立 → return -1 ✅

**触发右半段升序分支**：`nums=[6,7,0,1,2,4,5], target=5`（答案下标 6）：
- 轮1 `l=0,r=6,mid=3,nums[3]=1`：`nums[0]=6 <= 1`否 → **右半段** `[1,2,4,5]` 升序；`1<5<=5`是 → `l=4`
- 轮2、轮3 在右半段普通二分 → return 6 ✅

## 6. 解法步骤
1. `left=0, right=len-1`。
2. `while left <= right`：算 `mid`。
3. 命中 `return mid`；否则按"决策树"判断哪半段升序、target 在不在、往哪缩（指针一律 `mid+1`/`mid-1`）。
4. 循环结束 `return -1`。

## 7. 复杂度
- 时间 **O(log n)**：每轮排除一半。空间 O(1)。
- 注意：哪怕是这种"变形"二分，只要每轮稳定砍掉一半，就还是 O(log n)。

## 8. 易错点
- ⚠️ **指针只能 `mid+1` / `mid-1`，绝不能光秃秃 `=mid`**：否则区间缩不动 → `while` 死循环。（本题最初纠结"left 该不该=mid"就是漏了 ±1。）
- ⚠️ 判断 target 在不在升序半边，要把它**夹在区间里**（`nums[left] <= target < nums[mid]`），不能只比 `nums[mid]` 一个端点——否则 target 比整段还大/还小时会判错边。
- 边界取等：`nums[left] <= nums[mid]`（带等号，处理 left==mid 的单元素情况）。
- 区间端点用闭区间的两端 `nums[left]` / `nums[right]`，别写成 `nums[mid+1]` 之类。

## 9. 面试官追问 Q&A（每题必有）
- **Q：为什么用 `nums[left] <= nums[mid]` 判断哪半段升序，而不是去找断点？**
  A：找断点（最小值位置）要么 O(n) 扫一遍（违反 O(log n)），要么得额外写一轮二分（两次二分解法，更长）。直接在主二分里用 `nums[left] vs nums[mid]` 一步认出升序半边，一次二分就够，代码最短。
- **Q：为什么必须把 target 夹在区间里判断，不能只比 `nums[mid]`？**
  A：旋转数组里"比 mid 大"不代表"在 mid 右边"。只有先锁定一个**完整升序**的半段，"夹在 `[段首,段尾]` 里"才是 target 在该段的充要条件，逻辑才严密。
- **Q：如果数组里有重复元素（如 [2,2,2,0,2]）这套还成立吗？**
  A：不完全成立。当 `nums[left]==nums[mid]==nums[right]` 时无法判断哪半段升序，需要额外 `left+=1, right-=1` 跳过相等端点，最坏退化到 O(n)。那是 LeetCode 81 题。本题保证元素互不相同，所以这套干净的 O(log n) 成立。
- **Q：两次二分（先找旋转点再二分）和一次二分哪个好？**
  A：两次二分思路更直观（先 O(log n) 找最小值位置定位两段，再在对应段二分），但要写两段二分、处理旋转点边界，代码更长更易错。面试推荐这套一次二分。

## 10. 顺带建立的概念
- **"变形二分"的通用心法**：二分不一定要求整体有序，只要**每一步能稳定排除掉一半候选**就能用。旋转数组靠"总有一半升序"来做到这点。后面很多二分题（旋转最小值 153、二分答案 875）都是这个思想的延伸。
- 普通二分（704）是这题的特例：没旋转时整段都"升序半边"，决策树只会走问题2的"是"分支。
