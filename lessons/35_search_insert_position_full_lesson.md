## ① 读题
LeetCode 35 👉 https://leetcode.cn/problems/search-insert-position/
- 大白话：升序数组找 `target`，找到返回下标；找不到返回"它该插进去的位置"。
- 例 `nums=[1,3,5,6]`：`5`→`2`、`2`→`1`、`7`→`4`、`0`→`0`。
- 关键：和 704 代码 **99% 一样**，唯一区别在"找不到时返回什么"。

## ③ 你卡住的点 + 怎么讲通的（这题的核心）

### 卡点1：「没太懂，最后 left 比 right 大了？」
讲法：**对！最后 `left` 比 `right` 大 1，这正是循环该结束的标志，不是 bug。**
- `while left <= right`：只要还 `<=` 就有没查的数，继续；一旦 `left > right`，没得查了，停。
- `[left, right]` 框的是"还没排除、还要查的范围"，每查一次缩小，最后两指针交叉错开 1 → 范围空了 → 停。
- 演示 `target=2`：
  | 轮 | left | right | mid | nums[mid] | 动作 |
  |---|---|---|---|---|---|
  | 1 | 0 | 3 | 1 | 3>2 太大 | right=0 |
  | 2 | 0 | 0 | 0 | 1<2 太小 | left=1 |
  | 3 | 1 | 0 | — | left>right | 结束，return left=1 ✅ |

### 卡点2：「为什么直接 return left，我还是不能理解」（换角度才通）
**先把二分忘掉，想常识**：
> ⭐ **插入位置 = "比 target 小的数有几个"**
- 比 2 小的有 1 个（就是 1）→ 插下标 1；比 7 小的有 4 个 → 下标 4；比 0 小的 0 个 → 下标 0。

**再看 left 在二分里干嘛**：left 从 0 开始，只有 `nums[mid] < target` 时 `left = mid + 1`，每次都意味着"又确认一批比 target 小的数，跳到它们后面"。
- 所以循环结束时 `left` = "比 target 小的数的个数" = 插入位置。**return left 不是巧合，是 left 的工作本来就是数这个数。**

### 和 704 的唯一区别（一行）
```
704: 循环结束 → return -1
35:  循环结束 → return left
```
主体 `while`+三种情况和 704 一字不差。
