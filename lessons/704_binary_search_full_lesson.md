## ① 读题（带教给）
LeetCode 704 👉 https://leetcode.cn/problems/binary-search/
- 大白话：从小到大排好序的整数数组 `nums` + 目标 `target`。在就返回下标，不在返回 `-1`（无重复）。
- 例：`nums=[-1,0,3,5,9,12], target=9` → `4`；`target=2` → `-1`。
- 圈关键词：数组**有序**（这题的突破口）。

**完整三句话**（面试就这么讲）：①在有序数组里找一个数 → ②二分查找，每次看中间、砍掉一半、移动左右指针 → ③ O(log n)，每次范围减半。

## ③ 你卡住的点 + 怎么讲通的

### 卡点1：为什么是 O(log n)？（你问了两次说没懂）
讲法：**O(log n) = "n 能被砍一半几次，才砍到只剩 1 个"。**
```
16 → 8 → 4 → 2 → 1   数了 4 次
反过来：1→2→4→8→16  翻 4 次， 2 的 4 次方=16
所以"16 砍到 1 的次数" = "2 的几次方=16" = log₂16 = 4
```
| n | 一个个查 O(n) | 二分 O(log n) |
|---|---|---|
| 16 | 16 次 | 4 次 |
| 100万 | 100万次 | ~20 次 |
| 10亿 | 10亿次 | ~30 次 |
一句话：能反复对半砍的算法就是 O(log n)，超快。

### 代码骨架（带教给骨架，你填 3 种情况）
```python
while left <= right:
    mid = (left + right) // 2
    if nums[mid] == target: return mid
    elif nums[mid] < target: left = mid + 1
    else: right = mid - 1
return -1
```
你照思路把这 3 种情况填对了。

## ④ 跑通时踩的坑（重要）

### 坑1：bash 反斜杠被吃掉
你在终端跑 `python H:\leetcode\...\solution.py`，报错路径变成 `Hleetcodesolutions...704...`（**反斜杠 `\` 全被吃了**）。
- 原因：这是 bash，`\` 是转义符不是路径分隔符。
- 解决：bash 里路径一律用正斜杠 `/`，或给路径加引号。
> 规律：英文名 + 正斜杠，避开"中文乱码 + `\`被吃"两个坑。

### 顺带答你的问题：ACM 格式是不是从 import 开始写？
