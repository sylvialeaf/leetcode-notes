# 踩坑与基础概念（边学边攒）

## 一、终端 / 环境踩坑
1. **bash 终端路径用正斜杠 `/`，不要用 `\`**
   - 原因：bash 里 `\` 是"转义符"，会被吃掉，路径就拼错（`H:\leetcode\...` 变成 `Hleetcode...` 报 No such file）。
   - 正确：`python solutions/binary_search/704_binary_search/solution.py`；或给路径加引号 `"H:\..."`。
2. **文件名 / 文件夹名一律用英文（ASCII）**
   - 中文名在终端会乱码（编码不一致），复制含中文的命令粘贴回终端也乱码。
   - 中文只写在文件**内容**里（编辑器打开正常显示）。
3. **改完代码记得 `Ctrl + S` 保存**
   - 没保存的话，`python` 跑的还是磁盘上的旧文件。
   - 典型现象：函数体还是 `pass`、新代码没保存 → 运行输出一堆 `None`。

## 二、力扣模式 vs ACM 模式
| | 力扣模式（如 704） | ACM 模式（笔试） |
|---|---|---|
| 写什么 | 只写 `class Solution` 里的函数 | 完整程序：读输入 → 算 → 打印 |
| 谁喂数据 | 力扣框架帮你喂 | 自己从 stdin 读 |
- 阶段一全用力扣模式；ACM 在 `acm/` 文件夹专练。
- **IO** = 输入 / 输出；**EOF** = 输入结束（多组数据读到没有为止）。代码 `for line in sys.stdin:` 自动读到 EOF 停；手动测试时按 `Ctrl+Z` 回车（Windows）发一个 EOF。

## 三、Python 基础概念
1. **`pass`**：占位符，"什么都不做"。函数体不能空，没写逻辑时用它占位。函数只有 `pass`、没 `return` → 调用返回 `None`。
2. **三种"嵌套"别混**：
   - **循环嵌套** = `while` 里又有 `while`（二分这些题没有）。
   - **代码块嵌套** = `if` 写在 `while` 里面（靠缩进表示，正常结构）。
   - **函数调用** = 一个函数里调另一个函数 `self.xxx()`（不是循环嵌套）。
3. **`self.`**：调用"我自己这个类里"的其它方法 / 属性。
4. **提前返回（卫语句）**：先把特殊情况 `return` 掉，主逻辑更清晰（如 `if first == -1: return [-1,-1]`）。
5. **`mid` 是下标（在哪），`target` 是值（是几）**——要返回位置就记 `mid`，别记成 `target`。
6. **`if 函数():`**：函数返回 True/False 时，可直接 `if check(mid):`，意思是"check(mid) 返回 True 就进 if"。等价 `if check(mid) == True:`，但 `== True` 多余，不写更地道。`if` 后面跟的就是个"真/假表达式"。
7. **`math.ceil(x)` 向上取整**（天花板）：把数往大的方向进位到整数。`math.ceil(1.1)=2`、`math.ceil(2.0)=2`、`math.ceil(6/4)=2`。要先 `import math`。
   - 配套：`math.floor` 向下取整；普通整除 `//` 也是向下取整。要"零头也进 1"用 `ceil`。
   - 整数写法（无浮点误差、面试爱考）：`ceil(a/b)` == `(a + b - 1) // b`。
8. **`填空时把占位 `___` 删干净再写`**：写成 `mid + 1___` 会报 `SyntaxError: invalid decimal literal`（Python 把 `1___` 当成奇怪的数字）。
9. **模块名别写错**：`math.ceil` 不是 `miles.ceil`——`NameError: name 'xxx' is not defined` 多半是名字拼错或没 import。

## 四、复杂度速记 & 怎么算
### 量级直觉
- 量级：`O(1)` 常数 < `O(log n)` 对数（每次砍一半，超快）< `O(n)` 线性 < `O(n log n)` < `O(n²)`。
- `O(log n)` = "n 能对半砍几次砍到 1"。100 万 ≈ 20 步，10 亿 ≈ 30 步。
- 能"对半砍 / 每步排除一半"的算法，基本就是 `O(log n)`。

### 时间复杂度怎么算（三条规则）
看的是**"输入翻倍，步数怎么变"**，不是数代码行数。
1. **循环看次数**：扫一遍 `for x in nums` → `n` 次 → `O(n)`；每轮砍一半 → `log n` 次 → `O(log n)`。
2. **嵌套相乘**：循环套循环、或循环里调一个 O(n) 的函数 → 次数**相乘**。二分 log 次、每次做 O(n) → `O(n log n)`。
3. **扔常数留最高阶**：`O(2n)`→`O(n)`，`O(2 log n)`→`O(log n)`，`O(n + log n)`→`O(n)`。

### 空间复杂度怎么算
- 只看**额外**开的内存（输入本身不算）。只用几个变量（left/right/mid/hours）→ `O(1)`。
- 开了跟 n 等长的新数组、或递归 n 层 → `O(n)`。

### log 的底数
- 二分里 `log n` 其实是 `log₂n`（以 2 为底，对应"对半砍"）。
- 但大 O **忽略常数倍**，换底只差常数（`log₂n = log₁₀n / log₁₀2`），所以**写复杂度时不写底**，统一 `O(log n)`。算"砍几次"时心里用 2 为底即可。

### 二分专题 6 题复杂度对照
| 题 | 时间 | 空间 | 备注 |
|---|---|---|---|
| 704/35/33/153 | `O(log n)` | `O(1)` | 一个 while 砍一半 |

## 五、二分查找套路总结（做完 704/35/34/33/153/875 提炼）

### 两套模板（按"当前 mid 能不能被排除"选，选错会死循环）
**模板一：找特定值 / 找边界**（704/35/34/33）
- `while left <= right`，两边都 `mid ± 1`。
- 命中就 return，或指针交叉 `left > right` 结束。
```
left, right = 0, len-1
while left <= right:
    mid = (left+right)//2
    if 命中: return ...
    elif 偏小: left = mid + 1
    else:    right = mid - 1
```

**模板二：缩到单个幸存者 / 找第一个满足条件的**（153/875）
- `while left < right`，一边 `mid+1`、另一边 `right = mid`（mid 可能就是答案，不能排除）。
- 缩到 `left == right` 返回它。
```
left, right = 1, max
while left < right:
    mid = (left+right)//2
    if 可行/满足: right = mid     # 保留 mid
    else:        left = mid + 1
return left
```
> ⚠️ **`right = mid` 必须配 `while left < right`**；配 `<=` 会死循环。`mid ± 1` 配 `<=`。成对记。

### 各题"命中/结束时怎么处理"
- **找值（704）**：命中 `return mid`；找不到 `return -1`。
- **找插入位置（35）**：命中 `return mid`；找不到 `return left`（left = 比 target 小的数的个数）。
- **找第一个（34 左边界）**：命中 `ans=mid` 后 `right=mid-1`（往左压）。
- **找最后一个（34 右边界）**：命中 `ans=mid` 后 `left=mid+1`（往右压）。
- **旋转数组找值（33）**：先用 `nums[left]<=nums[mid]` 判哪半边升序，在升序半边判 target 是否**夹在区间里**决定缩哪边。指针仍 `mid±1`。
- **旋转数组找最小（153）**：比 `nums[mid]` 与 `nums[right]`，`mid>right` 则 `left=mid+1`，否则 `right=mid`。模板二。

### ⭐ 二分答案（875，重要升级）
不是在数组里找元素，而是**在"答案的取值范围(值域)"上二分**——数组可以完全无序，只要**答案对判定单调**就能用。
- **识别信号**："求最小/最大的 X 使得某条件成立"、"最小化最大值 / 最大化最小值"，答案有上下界且能 O(n) 验证。
- **三件套**：①答案范围（如速度 `[1, max(piles)]`）②判定函数 `check(x)`（O(n) 判断"行不行"，要对 x 单调）③在范围上二分找第一个/最后一个满足的值（用模板二）。
- 时间 `O(n · log(值域))`：别漏掉 check 每次的 O(n)。
- 后续同类高频题：1011 送包裹、410 分割数组等，同一招。

> 二分能举一反三的关键：骨架就两套，差别只在"命中时返回还是继续、往哪边压、能不能排除 mid"。
