## 三、小红书「本地 IDE 手撕练习」方法论（核心思路提炼）

> 来源：飞书文档《[生成式推荐] 大厂面试手撕算法本地 ide 练习步骤详解》

**核心理念**：在 IDE 里建一个"力扣练习工程"，把它同时当作 **个人题库** 和 **面试手撕环境**。

1. **创建项目**：像做正常项目一样建一个练习工程。
2. **分包分类**：按算法类型建包（heap / backTracking / dp / graph / linkedlist / tree / stack / slidingWindow / multiThread …）；另建一个 `Code`（scratch）类用于实时练习和面试共享屏幕手撕。
3. **自构建 case 模式（面试最常用‼️）**：
   - 每道题 = 一个类，**题目英文名 → 类名**，便于快速定位（如"第 k 大元素" → `HeapSort`）。
   - `main` 里写主流程：① 构建测试 case → ② 传参调核心方法 → ③ 处理输出。
   - 链表 / 二叉树额外写 `Node` / `TreeNode` 类，指定左右子节点。
   - **多解法**：同一类里写多个方法分别调用，对比时间/空间复杂度。
4. **ACM 模式**：把"构建样例"换成"自己读输入"，专练输入输出处理（笔试常用）。
5. **手撕题单范围**：Hot100、面试经典 150、多线程、单例模式（5 种写法）。少数企业自命题，掌握题单 + 举一反三即可。
6. **小技巧**：
   - 直接在面试用的电脑上练，熟悉环境；面试前一小时刷几道适应紧张感。
   - 建 **易忘 API 库**：各平台代码联想差异大（Word 手撕完全无联想），把常用类/方法签名整理成速查表，面试前常看常新。
   - **及时复盘**，标注每题掌握程度。

## 四、四类资源怎么配合用（一句话版）

- **代码随想录** 搭框架（我是谁、有哪些专题、每类套路是什么）。
- **灵茶山艾府** 练深度（某一类写不顺时的"心法 + 精选题"）。
- **Hot100 / 面试经典150** 是必刷主线题单。
- **CodeTop** 冲刺期按目标公司高频题查漏补缺。
- **本地 IDE 题库** 贯穿全程：把上面刷到的题沉淀成自己的、可手撕可复盘的题库。
