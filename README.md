## 一、学习方法（针对你的痛点）

### 痛点 1：总是忘 → 用「间隔重复 + 掌握度标注 + 记套路而非记答案」
- 每做完一题，在 [`progress/刷题记录.md`](progress/刷题记录.md) 里标 🔴/🟡/🟢 掌握度。
- 复盘节奏（艾宾浩斯）：做完后 **第 1 天 / 3 天 / 7 天 / 14 天** 重做 🔴🟡 的题。
- **重做时不看代码，只看文件顶部的「思路注释」，能否还原出代码** —— 还原不出就是没真懂。
- 记忆对象是**每一类的套路模板**（如"回溯=选择→递归→撤销选择"），不是某道题的具体代码。

### 痛点 2：ACM 模式不牢 → 单独开 `acm/` 专练输入输出
- 力扣是"核心代码模式"（只写函数），笔试是"ACM 模式"（自己读输入、打印输出）。
- 先背熟 [`acm/templates/io_template.py`](acm/templates/io_template.py) 的几套读入模板，再做 `acm/problems/` 里的题。
- 把"算法"和"输入输出"拆开练：算法在 `solutions/` 练，读写在 `acm/` 练。

### 痛点 3：没有体系化 → 按「专题」走，而不是随机刷
- **阶段一·搭框架**：跟 **代码随想录** 顺序，每个专题（数组→哈希→字符串→双指针→栈队列→树→回溯→贪心→DP→单调栈→图）系统过一遍，建立"看到题能归类"的能力。
- **阶段二·精练**：刷 **Hot100 + 面试经典150**；某类写不顺时看 **灵茶山艾府** 的精讲视频补"心法"。
- **阶段三·冲刺**：用 **CodeTop** 按目标公司刷高频题查漏补缺。
- 题库目录本身就是体系：`solutions/` 的分包 = 你的算法地图。

### 附加：说思路（面试要边写边讲）
- **三句话模板**：①「这题本质是 ___ 问题」→ ②「我用 ___ 方法，关键是 ___（数据结构/状态定义/不变量）」→ ③「时间 O(__) 空间 O(__)，因为 ___」。
- 每道题文件顶部的 `思路:` 注释就按这个模板写——**写得出注释 = 讲得出思路**。
- 进阶「一题三讲」：暴力解 → 为什么慢 → 怎么优化到最优。面试官爱听优化过程。
- 训练法：做完题后**对着思路注释，把解法大声复述一遍**（可录音回放），比默写代码更接近真实面试。

## 三、做一道新题的标准流程（自构建 case 模式）

1. 复制 `solutions/_TEMPLATE/` 整个文件夹 → 改名为 `<题号>_<英文名>`（如 `704_binary_search`）→ 放进对应分类文件夹。
2. 填 `problem.md`：原题 + LeetCode 链接。
3. 填 `solution.py`：顶部三句话思路 → 核心函数（多解法写 `solve_v2`）→ main 里①造 case ②调函数 ③打印（注上"期望: xx"）。
4. 运行 `python solutions\<分类>\<题号>_<英文名>\solution.py` 看结果对不对。
5. `notes.md` 记知识点（带教时我帮你整理）。
6. 在 `progress/刷题记录.md` 标掌握度，忘的 API 记进 `cheatsheet/python-api.md`。

> 已做的范例：`solutions/binary_search/704_binary_search/`（标准三件套结构）。
> 另有 3 个早期范例是旧的扁平单文件：`solutions/heap/kth_largest.py` 等，可运行参考，之后复习到再并入新结构。
